# Dev Tasks

Директория для заданий на разработку, подготовленных для использования в Cursor и/или Claude Code.

## Структура задания

```
[task-name]/
├── TASK.md           # Основное задание для AI (главный файл)
├── CONTEXT.md        # Контекст: архитектура, ограничения, стек
├── references/       # Референсы: примеры, спецификации, документация
│   └── ...
└── assets/           # Ресурсы: схемы, макеты, данные
    └── ...
```

## Workflow

### 1. Создание задания
```bash
# Скопировать шаблон
cp -r _template [task-name]
```

### 2. Заполнение
1. **TASK.md** — описать задачу, acceptance criteria, ограничения
2. **CONTEXT.md** — добавить контекст проекта (архитектура, зависимости)
3. **references/** — положить примеры кода, спецификации, API docs
4. **assets/** — схемы, макеты, тестовые данные

### 3. Использование в IDE
```bash
# Скопировать в проект
cp -r [task-name] /path/to/project/.claude/tasks/

# Или открыть как контекст в Cursor
# Cmd+Shift+P → "Add to context" → выбрать папку
```

### 4. После завершения
- Переместить в `archive/` или удалить
- Записать результаты/learnings в task если нужно

## Формат TASK.md

Главный файл, который читает AI. Оптимизирован для:
- **Cursor** — используется как context или в composer
- **Claude Code** — добавляется через /add или CLAUDE.md

Структура:
1. **Цель** — что нужно сделать (1-2 предложения)
2. **Контекст** — зачем это нужно
3. **Требования** — конкретные acceptance criteria
4. **Ограничения** — что НЕ делать, технические границы
5. **Референсы** — ссылки на файлы в references/

## Tips

- Держи TASK.md лаконичным — детали в CONTEXT.md и references/
- Используй чёткие acceptance criteria (чек-лист)
- Добавляй примеры желаемого результата когда возможно
- Указывай технологический стек явно
