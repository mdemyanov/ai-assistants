# Анатомия Skill

Детальное руководство по структуре и компонентам skill.

## Структура директории

```
skill-name/
├── SKILL.md              # Обязательно
│   ├── YAML frontmatter  # name, description (обязательно)
│   └── Markdown body     # Инструкции (обязательно)
└── Bundled Resources     # Опционально
    ├── scripts/          # Исполняемый код
    ├── references/       # Документация для контекста
    └── assets/           # Файлы для вывода
```

## SKILL.md

### Frontmatter (YAML)

**Обязательные поля:**

```yaml
---
name: skill-name
description: Comprehensive description with triggers
---
```

**Допустимые поля:**
- `name` — имя в hyphen-case (обязательно)
- `description` — описание с триггерами (обязательно)
- `license` — информация о лицензии
- `allowed-tools` — разрешённые инструменты
- `metadata` — дополнительные метаданные

**Правила name:**
- Только lowercase, цифры, дефисы
- Без дефисов в начале/конце
- Без двойных дефисов
- Максимум 64 символа

**Правила description:**
- Максимум 1024 символа
- Без угловых скобок `<>`
- Включает ЧТО делает + КОГДА использовать

### Body (Markdown)

Загружается ПОСЛЕ триггера skill. Содержит:
- Обзор (1-2 предложения)
- Workflow или инструкции
- Ссылки на ресурсы

**Ограничение:** <500 строк. При превышении → выноси в references/.

## Bundled Resources

### scripts/

**Назначение:** Исполняемый код для детерминированных операций.

**Когда добавлять:**
- Один и тот же код переписывается многократно
- Нужна детерминированная надёжность
- Операция хрупкая и требует точности

**Примеры:**
- `rotate_pdf.py` — поворот PDF
- `extract_form_fields.py` — извлечение полей формы
- `document.py` — модуль обработки документов

**Преимущества:**
- Экономия токенов
- Детерминированность
- Может исполняться без загрузки в контекст

### references/

**Назначение:** Документация для загрузки в контекст по необходимости.

**Когда добавлять:**
- Подробная документация для референса
- API спецификации
- Схемы баз данных
- Детальные гайды

**Примеры:**
- `schema.md` — схема базы данных
- `api_docs.md` — документация API
- `policies.md` — корпоративные политики
- `workflow_guide.md` — детальный гайд процесса

**Лучшие практики:**
- Если файл >10k слов → добавь паттерны grep в SKILL.md
- Не дублируй информацию из SKILL.md
- Структурируй для быстрого поиска

### assets/

**Назначение:** Файлы для использования в выводе (не в контексте).

**Когда добавлять:**
- Шаблоны документов
- Изображения, иконки
- Boilerplate код
- Шрифты

**Примеры:**
- `template.pptx` — шаблон презентации
- `logo.png` — логотип
- `hello-world/` — boilerplate проект
- `font.ttf` — шрифт
