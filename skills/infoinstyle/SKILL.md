# Инфо в стиль

Трансформация текстов: убрать ИИ-паттерны → применить инфостиль → добавить жизнь.

## Core Principles

### 1. Диагностика важнее правки
Сначала понять, какие именно паттерны портят текст, потом чинить.

### 2. Конкретика вместо абстракций
Факт > мнение. Цифра > оценка. Пример > объяснение.

### 3. Сохранять авторский голос
Инфостиль — не стерилизация. Убирать мусор, сохранять характер.

---

## Определение режима

| Режим | Когда | Что делать |
|-------|-------|------------|
| **Quick** | Мелкие правки, срочно | Только критичные паттерны (канцелярит, вода) |
| **Standard** | Большинство текстов | Полный workflow: диагностика → трансформация |
| **Deep** | Полная переработка | + структурные изменения, пирамида |

Если режим неясен из запроса — используй **Standard**.

## Workflow

### 1. Контекст (если нужно)

Уточни **только недостающее**:

| Вопрос | Когда спрашивать |
|--------|------------------|
| Цель текста | Если неясна (продать / информировать / убедить) |
| Аудитория | Если влияет на тон (специалисты / широкая) |
| Ограничения | Если есть (длина, формат, tone of voice) |

Если контекста достаточно — сразу к диагностике.

### 2. Диагностика

Найди паттерны по 4 категориям:

| Категория | Маркеры | Критичность |
|-----------|---------|-------------|
| **Content** | «В современном мире...», «Важно понимать...», списки ради списков | Высокая |
| **Language** | «В рамках», «является», «осуществлять», пассивный залог | Высокая |
| **Style** | Одинаковая длина предложений, дидактичный тон, перестраховки | Средняя |
| **Communication** | «Прежде всего...», «Подводя итог...», эхо вопроса | Средняя |

> Полный справочник: `references/ai-patterns.md`

### 3. Трансформация

Применяй последовательно:

1. **Удаление** — стоп-слова, мусорные связки, преамбулы
2. **Замена** — канцелярит → простой язык
3. **Усиление** — абстракции → конкретика (факты, цифры, примеры)
4. **Активация** — пассив → актив, отглагольные → глаголы
5. **Ритмизация** — разнообразить длину предложений

> Детали: `references/ilyahov-rules.md`

### 4. Проверка

Чеклист перед выдачей:

- [ ] Главное в начале?
- [ ] Нет предложений >2 строк?
- [ ] Есть конкретика (цифры, примеры)?
- [ ] Тон соответствует задаче?
- [ ] Нет канцелярита и стоп-слов?
- [ ] Авторский голос сохранён?

## Формат ответа

### Режим Standard / Deep

```markdown
## Диагностика
[2-4 главных проблемы по категориям]

## Ключевые изменения
[3-5 главных правок с кратким обоснованием]

## Результат
[Отредактированный текст]

## Что ещё можно улучшить
[Опционально: что выходит за рамки текущей правки]
```

### Режим Quick

```markdown
[Отредактированный текст]

---
Убрано: [кратко что изменилось]
```

### Режим Analysis Only

Если пользователь просит **только анализ** без редактирования:

```markdown
## Диагностика

### Content
[Найденные проблемы]

### Language
[Найденные проблемы]

### Style
[Найденные проблемы]

### Communication
[Найденные проблемы]

## Рекомендации
[Что исправить в первую очередь]
```

---

## Антипаттерны

- **НЕ** превращай живой авторский стиль в стерильный текст
- **НЕ** удаляй эмоции, если они уместны для задачи
- **НЕ** делай все предложения одинаково короткими (нужен ритм)
- **НЕ** убирай терминологию без замены на понятный аналог
- **НЕ** добавляй «конкретику» если у тебя нет реальных данных
