# Prompt Creator

Создание высокоэффективных промтов через 4-этапный процесс: понимание → проектирование → реализация → валидация.

## Core Principles

### Claude уже умён

Не объясняй то, что Claude знает из обучения. Промт — это delta, а не полная инструкция.

**Добавляй только:**
- Специфику предметной области (термины, стандарты, ограничения)
- Желаемый формат вывода
- Критерии успеха для конкретной задачи
- То, чего Claude НЕ должен делать

### Degrees of Freedom

Выбирай уровень детализации промта по критичности задачи:

| Уровень | Когда | Формат промта |
|---------|-------|---------------|
| Высокий | Творческие задачи, много валидных подходов | Роль + цель + ограничения |
| Средний | Стандартные бизнес-задачи | + процесс + формат вывода |
| Низкий | Критичные операции, консистентность важна | + примеры + чеклисты + валидация |

### Цель — в начале

Первые 2-3 предложения промта определяют всё остальное. Claude использует их как якорь для интерпретации остальных инструкций.

## Workflow

### 1. Понимание задачи (обязательно)

Собери конкретику перед созданием:

| Вопрос | Зачем | Минимум |
|--------|-------|---------|
| Какую задачу решает? | Определяет цель и критерии успеха | ✅ Обязательно |
| Кто аудитория? | Влияет на тон, глубину, терминологию | ✅ Обязательно |
| Контекст использования? | Определяет ограничения | По возможности |
| Примеры результата? | Лучший способ понять ожидания | По возможности |

**Цель:** Чёткое понимание use case перед проектированием.

### 2. Проектирование

#### Выбери структуру промта

| Структура | Когда использовать | Секции |
|-----------|-------------------|--------|
| **Минимальная** | Простые задачи, опытные пользователи | Роль + Цель + Формат |
| **Стандартная** | Большинство задач | + Контекст + Процесс + Границы |
| **Расширенная** | Сложные/критичные задачи | + Примеры + Валидация + Адаптация |

#### Выбери техники

| Тип задачи | Техники |
|------------|---------|
| Аналитическая | Chain-of-Thought, Self-Reflection |
| Творческая | Role-Playing, Few-Shot |
| Консультативная | Structured Output, Constraints |
| Обучающая | Scaffolding, Progressive Disclosure |

> Детали техник: `references/techniques.md`

### 3. Реализация

Используй шаблон из `assets/prompt-template.md`.

**Чеклист качества:**
- [ ] Цель понятна из первых 2-3 предложений
- [ ] Нет размытых слов ("хорошо", "качественно") без критериев
- [ ] Есть примеры для нестандартных форматов
- [ ] Явные границы (что НЕ делает)
- [ ] Нет противоречий между секциями

### 4. Валидация

Прогони мысленно 3 тест-кейса:

| Тип | Проверяет |
|-----|-----------|
| Типичный запрос | Работает ли основной сценарий? |
| Граничный случай | Как обрабатывается неопределённость? |
| Off-topic запрос | Есть ли защита от нецелевого использования? |

## Формат ответа

```markdown
## Понимание задачи
[Use case и аудитория — 2-3 предложения]

## Проектирование
- Структура: [минимальная/стандартная/расширенная]
- Техники: [список с кратким обоснованием]

## Промт

[Полный текст]

## Тестовые сценарии
1. [Типичный]
2. [Граничный]
3. [Off-topic]
```

## Интеграции

### CLAUDE.md (Project Configuration)

Для промтов, работающих с codebase, создавай `CLAUDE.md` в корне проекта:

```markdown
# Project Context

## Архитектура
[Краткое описание структуры проекта]

## Стандарты кода
[Coding conventions, линтеры, форматтеры]

## Workflow
[Как запускать тесты, билд, деплой]
```

**Когда нужен CLAUDE.md:**
- Промт работает с конкретным репозиторием
- Нужен постоянный контекст проекта
- Команда использует Claude Code

### Делегирование к Skills

Промт может делегировать задачи к специализированным skills:

```markdown
## Делегирование

При получении запроса на создание skill:
1. Загрузи `skills/nau-skill-creator/SKILL.md`
2. Следуй workflow из skill

При получении запроса на анализ промта:
1. Загрузи `skills/prompt-review/SKILL.md`
2. Примени чеклист оценки
```

**Когда делегировать:**
- Задача требует специализированного workflow
- Skill уже реализует нужную функциональность
- Нужна консистентность с другими ассистентами

## Антипаттерны

- ❌ Объяснение того, что Claude и так знает
- ❌ Размытые формулировки без критериев ("будь полезным")
- ❌ Противоречивые инструкции
- ❌ Промт >2000 слов без разделения на секции
- ❌ Отсутствие примеров для сложных форматов
- ❌ Требования, которые Claude не может выполнить
