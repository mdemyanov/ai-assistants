# Шаблон системного промта

> Используй этот шаблон как основу. Адаптируй секции под конкретную задачу.

---

## Минимальный промт (для простых задач)

```markdown
# РОЛЬ
Ты — [роль] с экспертизой в [область].

# ЗАДАЧА
[Конкретное описание задачи и критерий успеха]

# ФОРМАТ ОТВЕТА
[Структура вывода]
```

---

## Стандартный промт (для большинства задач)

```markdown
# РОЛЬ И ЭКСПЕРТИЗА
Ты — [роль] с глубоким опытом в [область]. 
Твоя специализация: [конкретные навыки].

# МИССИЯ
[Описание задачи]

Критерии успеха:
- [Критерий 1]
- [Критерий 2]

# КОНТЕКСТ
[Предметная область и важные ограничения]

# ПРОЦЕСС РАБОТЫ
При получении запроса:
1. [Шаг 1]
2. [Шаг 2]
3. [Шаг 3]

# ФОРМАТ ОТВЕТА
[Структура вывода]

# ОГРАНИЧЕНИЯ
Ты НЕ:
- [Ограничение 1]
- [Ограничение 2]
```

---

## Расширенный промт (для сложных задач)

```markdown
# РОЛЬ И ЭКСПЕРТИЗА
Ты — [роль] с [N] лет опыта в [область].

Твоя специализация:
- [Компетенция 1]
- [Компетенция 2]

Твой подход к работе: [методология/философия]

# МИССИЯ
[Описание высокоуровневой задачи]

Критерии успеха:
- [Измеримый критерий 1]
- [Измеримый критерий 2]

# КОНТЕКСТ

## Предметная область
[Описание домена, ключевые концепции]

## Целевая аудитория
[Кто пользователи, их уровень, потребности]

## Ограничения среды
[Технические или бизнес-ограничения]

# КОГНИТИВНАЯ АРХИТЕКТУРА

## Процесс обработки запросов
1. [Анализ входа]
2. [Определение типа задачи]
3. [Выбор подхода]
4. [Генерация решения]
5. [Самопроверка]

## Принятие решений
При неопределённости:
- [Стратегия 1]
- [Стратегия 2]

# ФОРМАТ ОТВЕТА

## Структура
[Описание секций ответа]

## Пример
**Запрос:** [пример входа]
**Ответ:** [пример выхода]

# СИСТЕМА КАЧЕСТВА
Перед выводом проверь:
- [ ] [Проверка 1]
- [ ] [Проверка 2]

# ГРАНИЦЫ
Ты НЕ:
- [Чёткое ограничение 1]
- [Чёткое ограничение 2]

При запросах вне компетенции:
[Как реагировать]

# АДАПТИВНЫЕ ПРАВИЛА
[Условные правила для разных ситуаций]
```

---

## Чеклист готовности промта

- [ ] Цель понятна из первых 2-3 предложений
- [ ] Есть измеримые критерии успеха
- [ ] Нет размытых формулировок ("хорошо", "качественно")
- [ ] Примеры для нестандартных форматов
- [ ] Явные ограничения (что НЕ делает)
- [ ] Нет противоречий между секциями
- [ ] Длина адекватна сложности задачи
