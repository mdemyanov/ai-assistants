## Пользователь

### Интересы

- Быстрое решение рабочих вопросов без погружения в технические детали
- Прозрачность статуса своих обращений
- Минимум усилий для получения помощи
- Доступ к самообслуживанию для типовых запросов

### Боли и проблемы

- Непонятно, куда обращаться с конкретной проблемой
- Долгое ожидание ответа без информации о прогрессе
- Необходимость повторно объяснять проблему разным специалистам
- Сложные формы заявок с непонятными полями
- Отсутствие истории предыдущих обращений

### Выгоды

- Единая точка входа для всех обращений
- Автоматическое информирование о статусе
- Быстрый доступ к базе знаний для самостоятельного решения
- История всех взаимодействий с поддержкой

## Работа

### Роль

Сотрудник организации, использующий ИТ-сервисы для выполнения своих основных рабочих задач. Не является техническим специалистом. Взаимодействует с системой поддержки эпизодически — при возникновении проблем или потребности в услугах.

## Решение

### Где можно помочь

- Интеллектуальная маршрутизация заявок без участия пользователя
- Чат-бот для типовых вопросов и первичной диагностики
- Автозаполнение форм на основе контекста (локация, оборудование)
- Проактивные уведомления о массовых инцидентах
- Персонализированный каталог услуг

### Что можно не делать

- Глубокая техническая диагностика (передаётся специалистам)
- Настройка сложных интеграций
- Управление процессами поддержки

### Что получит пользователь

- Экономия времени на оформление и отслеживание заявок
- Уверенность, что проблема решается
- Возможность работать, не отвлекаясь на технические вопросы
- Прозрачная коммуникация с поддержкой
