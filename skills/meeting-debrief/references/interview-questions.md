# Уточняющие вопросы

Структурированные вопросы для извлечения информации о встрече через диалог.

## Принципы

**Минимум вопросов.** Задавай только то, что нельзя извлечь из источника.

**Конкретика.** Не «как прошла встреча?», а «какие решения приняли?».

**Прогрессивное уточнение.** Сначала общие вопросы, затем детали по ответам.

## Базовый набор (интервью с нуля)

Когда нет транскрипта или переписки — только диалог с пользователем.

### 1. Контекст

```
С кем была встреча и когда?
```

### 2. Ключевые результаты

```
Какие ключевые решения были приняты на встрече?
```

### 3. Договорённости

```
Какие договорённости достигнуты?
Для каждой: кто отвечает и какой срок?
```

### 4. Задачи

```
Какие задачи появились по результатам?
Кто владелец каждой задачи?
```

### 5. Открытые вопросы

```
Что осталось нерешённым или требует follow-up?
```

### 6. Изменения

```
Изменились ли цели или приоритеты [участника]?
```

### 7. Наблюдения

```
Есть ли что-то важное, что стоит зафиксировать для себя?
(Настроение, сигналы, риски, идеи)
```

## Уточняющие вопросы (после анализа источника)

Когда транскрипт/переписка есть, но информация неполная.

### По договорённостям

| Ситуация | Вопрос |
|----------|--------|
| Есть действие, нет владельца | «Кто отвечает за [действие]?» |
| Есть действие, нет срока | «Какой срок по [действие]?» |
| Размытая формулировка | «Что конкретно имелось в виду под [цитата]?» |
| Неясно, решение или обсуждение | «[Цитата] — это финальное решение или ещё обсуждается?» |

### По участникам

| Ситуация | Вопрос |
|----------|--------|
| Неизвестный спикер | «Кто такой [Имя] в транскрипте?» |
| Несколько людей с одним именем | «Какой именно [Имя] имеется в виду?» |

### По контексту

| Ситуация | Вопрос |
|----------|--------|
| Упомянут проект без деталей | «О каком проекте шла речь, когда обсуждали [тема]?» |
| Ссылка на прошлые договорённости | «Какие предыдущие договорённости имелись в виду?» |

## Формат задавания вопросов

### Единичный вопрос

```markdown
Уточнение по встрече:

В транскрипте упоминается задача «проработать границы», но не указан срок.

**Вопрос:** Какой срок для этой задачи?
```

### Несколько вопросов

```markdown
Несколько уточнений по встрече:

1. Кто отвечает за подготовку документа?
2. Какой срок по задаче «согласовать с HR»?
3. «Встретимся на следующей неделе» — конкретный день есть?

Можете ответить списком или по одному.
```

### Подтверждение извлечённого

```markdown
Проверьте, правильно ли я понял:

**Договорённости:**
1. Ольга → прислать документ → до пятницы
2. Сева → проверить метрики → эта неделя

**Решения:**
- Идём по варианту А с дизайн-системой

Всё верно? Что-то упущено?
```

## Вопросы по типам встреч

### 1-1 с подчинённым

```markdown
Дополнительно по 1-1:

1. Есть ли обратная связь, которую нужно зафиксировать?
2. Изменился ли статус по его целям?
3. Какие-то сигналы по настроению/мотивации?
```

### Проектная встреча

```markdown
Дополнительно по проекту:

1. Изменился ли статус проекта? (on track / at risk / blocked)
2. Появились ли новые риски?
3. Сдвинулись ли сроки?
```

### Комитет

```markdown
Дополнительно по комитету:

1. Какие решения требуют оформления/коммуникации?
2. Есть ли action items для тебя лично?
3. Нужен ли follow-up с кем-то из участников?
```

## Когда НЕ задавать вопросы

- Информация уже есть в источнике
- Вопрос можно решить по контексту
- Детали не влияют на фиксацию (имена, которые можно найти в vault)
- Пользователь уже дал достаточно информации

## Эскалация при недостатке данных

Если после вопросов информации всё ещё недостаточно:

```markdown
Для полноценной фиксации не хватает информации по:
- [пункт 1]
- [пункт 2]

Варианты:
1. Зафиксировать то, что есть (частично)
2. Вернуться к источнику (перечитать переписку, запросить запись)
3. Отложить обработку

Как поступим?
```
