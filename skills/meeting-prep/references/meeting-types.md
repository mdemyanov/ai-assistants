# Специфика типов встреч

Ключевые отличия в подготовке. Общие принципы встреч Claude знает — здесь только специфика.

## 1-1 с подчинённым

**Главное:** Это ЕГО время, не твоё.

**Структура:**
1. Check-in
2. Review договорённостей
3. **Его темы** (приоритет!)
4. Твои темы
5. Блокеры
6. Договорённости

**Специфика подготовки:**
- Договорённости с прошлой встречи
- Прогресс по его целям
- Вопросы по его подчинённым (если есть)
- Обратная связь (что сказать)

**Red flags (для deep):**
- Нет обратной связи несколько встреч
- >70% времени говоришь ты
- Избегание сложных тем

**Вопросы-помощники:**
- Что больше всего мешает работать?
- Как я могу помочь?
- Есть что-то, о чём мы не говорим, но стоило бы?

---

## 1-1 с руководителем / равным

**Главное:** Приходи с решениями, не только проблемами.

**Структура:**
1. Check-in (короткий)
2. **Твои темы** (ты инициатор)
3. Его темы
4. Договорённости

**Специфика подготовки:**
- Чёткие запросы с обоснованием
- Готовые решения, варианты
- Аргументация для спорного

---

## Проектная встреча

**Главное:** Статус → Блокеры → Решения → Следующие шаги.

**Структура:**
1. Статус (где мы)
2. Прогресс с прошлой встречи
3. Блокеры и решения
4. Следующие шаги (кто, что, когда)
5. Риски

**Специфика подготовки:**
- INDEX.md проекта — актуальный статус
- Список вопросов для решения
- По каждой задаче: on track / at risk / blocked

**Red flags:**
- Задачи без владельца
- Решения без сроков
- Блокеры без плана

## Комитет / Стратсессия

**Главное:** Позиция + Аргументы + Варианты.

**Структура:**
1. Контекст (почему собрались)
2. Темы повестки с вариантами
3. Обсуждение
4. Решения (фиксировать!)
5. Action items

**Специфика подготовки:**
- Своя позиция по каждому вопросу
- Аргументы с данными
- Ответы на возможные возражения
- Альтернативы, если основное не пройдёт

**Формат темы:**
```markdown
### [Тема]

**Контекст:** [почему важно]

**Варианты:**
1. [А] — плюсы, минусы
2. [Б] — плюсы, минусы

**Рекомендация:** [что предлагаю]
```

## Встреча с заказчиком / партнёром

**Главное:** Управляй ожиданиями. Не обещай невозможное.

**Структура:**
1. Повестка (согласовать в начале)
2. Статус обязательств
3. Демо (если есть)
4. Риски (проактивно!)
5. Следующие шаги

**Специфика подготовки:**
- Что обещали → что сделали
- Риски и план Б
- Границы (что вне scope)

**Red flags:**
- Согласие на нереалистичное
- Риски не озвучены
- Нет письменной фиксации

## Универсальный чеклист

Для любой встречи:

- [ ] Ясна цель (что должно измениться после)
- [ ] Изучены материалы / история
- [ ] Есть список тем / вопросов
- [ ] Понятны ожидания участников
- [ ] Готов способ фиксации решений
