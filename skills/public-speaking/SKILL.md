# Public Speaking Prep

Подготовка эффективных публичных выступлений через 4-этапный процесс: понимание → структурирование → аргументация → подача.

## Core Principles

### Ключевая идея — основа всего

Слушатель должен унести ОДНУ главную мысль. Всё остальное — поддержка этой мысли. Если не можешь сформулировать ключевую идею в одном предложении — выступление не готово.

### Аудитория определяет содержание

Один и тот же материал подаётся по-разному для разных аудиторий. Примеры, терминология, глубина — всё адаптируется под слушателей.

### Структура важнее контента

Хорошо структурированное среднее выступление лучше хаотичного блестящего. Люди запоминают структуру, а не отдельные факты.

## Degrees of Freedom

| Уровень | Когда | Что делаем |
|---------|-------|------------|
| **Экспресс** | <10 минут, знакомая тема | Ключевая идея + 3 тезиса + call-to-action |
| **Стандарт** | 10-30 минут, важное выступление | + полная структура + аргументация + примеры |
| **Глубокий** | Keynote, критичная презентация | + работа с возражениями + визуализация + репетиция |

## Workflow

### 1. Понимание контекста

Минимум для начала работы:

| Вопрос | Зачем |
|--------|-------|
| Тема и повод? | Определяет фокус |
| Какую мысль должны унести? | Ключевая идея |
| Кто аудитория? | Адаптация контента |
| Сколько времени? | Объём и глубина |
| Какой результат нужен? | Цель: убедить/информировать/мотивировать/побудить к действию |

> Расширенные вопросы для глубокой проработки: `references/preparation-questions.md`

### 2. Структурирование

**Универсальная структура:**

```
ВВЕДЕНИЕ (10-15% времени)
├── Захват внимания (история/вопрос/факт)
├── Обозначение темы и актуальности
└── Анонс структуры

ОСНОВНАЯ ЧАСТЬ (70-80% времени)
├── Тезис 1 → Обоснование → Пример → Мини-вывод
├── Тезис 2 → Обоснование → Пример → Мини-вывод
└── Тезис 3 → Обоснование → Пример → Мини-вывод

ЗАКЛЮЧЕНИЕ (10-15% времени)
├── Резюме ключевой идеи
└── Призыв к действию
```

Оптимально 3-4 тезиса. Больше — аудитория не запомнит.

> Детали структуры с примерами: `references/speech-structure.md`

### 3. Проработка аргументации

Каждый тезис по формуле:

1. **Тезис** — чёткое утверждение
2. **Обоснование** — почему это так (логика, данные, авторитет)
3. **Иллюстрация** — пример, история, метафора, аналогия
4. **Мини-вывод** — связь с ключевой идеей

Примеры и истории подбираются под аудиторию. IT-специалистам — технические кейсы. Руководителям — бизнес-результаты.

### 4. Рекомендации по подаче

Для уровня "Стандарт" и "Глубокий":
- Интерактив: где задать вопросы, где сделать паузу
- Визуализация: что показать на слайдах
- Невербалика: ключевые моменты для акцентов

> Детальные техники подачи: `references/delivery-techniques.md`

## Формат вывода

### Экспресс
```markdown
## Ключевая идея
[Одно предложение]

## Структура (3 тезиса)
1. [Тезис] — [пример в одно предложение]
2. [Тезис] — [пример]
3. [Тезис] — [пример]

## Call-to-action
[Что должны сделать слушатели]
```

### Стандарт
```markdown
## Контекст
- Аудитория: [кто]
- Цель: [убедить/информировать/мотивировать]
- Время: [X минут]

## Ключевая идея
[Одно предложение]

## План выступления

### Введение
[Захват внимания + анонс]

### Основная часть

#### Тезис 1: [формулировка]
- Обоснование: [почему]
- Пример: [история/данные]
- Мини-вывод: [связь с ключевой идеей]

[...тезисы 2-3...]

### Заключение
[Резюме + призыв к действию]

## Рекомендации по подаче
[2-3 конкретных совета]
```
