# Паттерны эффективной работы с AI-ассистентом

## 1. Обработка встреч

### Транскрипт → Структура → Действия

**Алгоритм:**
1. Записать встречу (любой инструмент транскрипции)
2. Загрузить транскрипт в Claude
3. Запрос:

```
Обработай транскрипт [тип встречи] с [участники] от [дата].

Формат:
1. Ключевые темы (макс 5)
2. Принятые решения с контекстом
3. Договорённости: кто, что, когда
4. Вопросы для следующей встречи
5. Наблюдения для себя

Сохрани по шаблону [шаблон] в [путь].
```

---

## 2. Управление людьми

### Подготовка к 1-1

**Запрос (за 15-30 минут до встречи):**

```
Подготовь меня к 1-1 с [имя]:

1. Профиль: роль, текущие задачи
2. История: ключевые темы из последних 3 встреч
3. Открытые договорённости
4. Темы для обсуждения
5. Вопросы, которые стоит задать
6. Предупреждения (просрочки, конфликты)
```

**Важно:** AI должен честно указывать на ваши невыполненные обещания.

### Обновление профиля после встречи

Что фиксировать:
- Изменения в роли/зоне ответственности
- Новые цели
- Сильные стороны и зоны развития
- Важный контекст (мотивация, стиль коммуникации)

## 3. Принятие решений

### Структурированный анализ

**1. Описание ситуации:**
```
Контекст: [что происходит]
Проблема: [что нужно решить]
Ограничения: [что нельзя нарушать]
Стейкхолдеры: [кого затрагивает]
Уже пробовали: [что не сработало]
```

**2. Запрос вариантов:**
```
Предложи 3 варианта. Для каждого:
- Суть подхода
- Плюсы / Минусы / Риски
- Что потребуется для реализации
```

**3. Критика выбора:**
```
Я склоняюсь к варианту [X]. Покритикуй:
- Что я могу упускать?
- Какие риски недооцениваю?
- Кто будет против и почему?
```

**4. Фиксация в формате ADR**

## 4. Управление знаниями

### Еженедельный review

**Запрос (в конце недели, 15-20 минут):**

```
Weekly review:

1. Какие встречи были на этой неделе?
2. Какие задачи завершены?
3. Какие договорённости появились?
4. Что просрочено или под угрозой?
5. Топ-5 приоритетов на следующую неделю
6. Кому я забыл ответить?

Обнови 06_CURRENT/priorities.md
```

### Регулярные обновления

| Что обновлять | Когда | Триггер |
|---------------|-------|---------|
| priorities.md | Еженедельно | Review недели |
| profile.md | После 1-1 | Новая информация |
| 05_DECISIONS/ | При решении | Значимое решение |
| 01_CONTEXT/ | Ежемесячно | Изменения роли |

## 5. Работа с AI: практические советы

### Как добиться критики, а не согласия

**Явно просите:**
```
Покритикуй это решение. Что я упускаю?
```

**Адвокат дьявола:**
```
Представь, что ты против этого решения. 
Какие аргументы приведёшь?
```

**Фиксируйте в промте:**
```
Принцип: Критическое мышление важнее согласия.
```

### Как получать конкретику

**Указывайте контекст:**
```
Учитывая специфику [компания], [рынок], [ограничения]...
```

**Требуйте измеримости:**
```
Конкретные метрики, сроки, имена ответственных.
```

**Отвергайте абстракции:**
```
Это слишком общо. Что именно сделать завтра?
```

### Работа с длинным контекстом

1. **Начинайте с загрузки контекста:** role_scope.md, constraints.md
2. **Периодически напоминайте:** ключевые ограничения
3. **Разбивайте на сессии:** сложные задачи — отдельными диалогами
4. **Фиксируйте результаты:** сохранять в файлы

## 6. Антипаттерны

| Антипаттерн | Проблема | Решение |
|-------------|----------|---------|
| AI как исполнитель | Только написание текстов | AI — советник для решений |
| Слепое доверие | Принимать всё от AI | Критически оценивать |
| Одноразовые диалоги | Потеря контекста | Фиксировать в базе |
| Переусложнение | Сложная структура | Начать с минимума |
| Идеальный промт | Часы на "идеальный" | Итеративно улучшать |

## 7. Регулярные ритуалы

| Ритуал | Частота | Время | Что делать |
|--------|---------|-------|------------|
| Обработка 1-1 | После встречи | 10-15 мин | Транскрипт → структура |
| Подготовка к 1-1 | Перед встречей | 10 мин | Бриф от AI |
| Weekly review | Пятница | 20 мин | Статус, приоритеты |
| Обновление контекста | Ежемесячно | 30 мин | role_scope, constraints |
