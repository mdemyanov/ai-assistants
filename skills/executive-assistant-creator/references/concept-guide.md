# Концепция AI-ассистента руководителя

## Проблема

Руководитель ежедневно:
- Принимает десятки решений в условиях неполной информации
- Переключается между стратегическими и операционными задачами
- Теряет контекст при переключении между проектами и людьми
- Не имеет времени на систематизацию знаний

**Следствие:** важная информация теряется, решения принимаются без учёта контекста.

## Решение

Персональный AI-ассистент, который:
- **Помнит** весь контекст работы (люди, проекты, решения)
- **Структурирует** информацию в удобном для поиска виде
- **Советует** с учётом ограничений и принципов
- **Критикует** идеи, а не соглашается со всем

## Принципы организации базы знаний

### Разделение контекста и контента

**Контекст (01_CONTEXT)** — редко меняется:
- Роль и полномочия
- Ограничения и "красные линии"
- Ключевые стейкхолдеры

**Контент (остальные папки)** — часто меняется:
- Текущие задачи
- Записи встреч
- Принятые решения

### Frontmatter для метаданных

Каждый файл начинается с YAML-блока:

```yaml
---
type: person | 1-1 | decision | task | note
status: active | done | planned
date: 2025-12-01
tags: [tag1, tag2]
---
```

Это позволяет:
- Автоматически строить дашборды (Dataview)
- Фильтровать и искать документы
- AI понимает тип документа

### Связи между документами

Вики-ссылки `[[название]]` для связывания:
- Профиль человека ← Записи 1-1
- Решение ← Задачи, которые оно породило
- Проект ← Участники, решения

## Иерархия источников

При конфликте информации приоритет:
1. **Явные указания в диалоге** — высший приоритет
2. **Файлы базы знаний** — актуальный контекст компании
3. **Знания модели** — общие практики, фреймворки

## Этапы внедрения

### Этап 1: Минимальный старт (1-2 недели)
- Установить Obsidian + Claude Desktop + MCP
- Создать базовую структуру (01_CONTEXT, 04_TEMPLATES, 07_PEOPLE)
- Написать системный промт
- Начать вести записи 1-1

### Этап 2: Наращивание контекста (1-2 месяца)
- Наполнить профили людей
- Накопить историю встреч и решений
- Настроить Dataview-дашборды
- Отточить промт под свой стиль

### Этап 3: Автоматизация рутины (2-3 месяца)
- Автоматическая обработка транскриптов
- Регулярные review недели/месяца
- Интеграция с другими инструментами

## Типичные ошибки

1. **Слишком сложная структура с самого начала** — начните с минимума
2. **AI как исполнитель, а не советник** — требуйте критики
3. **Не обновлять базу знаний** — встроить в регулярные ритуалы
4. **Слишком длинный системный промт** — детали в файлы
5. **Не проверять факты от AI** — критически оценивайте ответы
